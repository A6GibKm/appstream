A component is wrapper around a metainfo.xml file or previously an appdata.xml file. It describes an application to the various stores out there on Linux.

Fields

kind: ComponentKind

The component type.

id: AppId

Unique identifier for this component.

name: TranslatableString

A human-readable name.

A short summary of the component.

The license of the compoonent.

metadata_license: Option<License>

The license the metainfo XML file released under.

project_group: Option<String>

Identify the project with a specific upstream umbrella. Known values includes: GNOME, KDE, XFCE, MATE, LXDE.

compulsory_for_desktop: Option<String>

Indicate for which desktop environment the component is essential for its functionality.

extends: Vec<AppId>

The various AppId that the current component extends.

The developers or the projects responsible for the development of the project.

Used by distributors to contact the project. The information should not be exposed to the user.
